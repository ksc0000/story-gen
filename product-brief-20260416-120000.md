## 1. プロダクトビジョン【PdM視点】

### 1.1 解決する課題

1. **既製絵本では子どもの「いま」に合わせた話を作れない** — 子どもの名前・好きなキャラクター・体験談を盛り込んだオリジナル絵本を作る手段がない
2. **手作り絵本はハードルが高い** — 絵が描けない・時間がない親でも、プロ品質に近い絵本を作りたいニーズがある
3. **既存AI生成ツールは子ども向けコンテンツ作成に不向き** — 安全性フィルタが不明瞭、UI が複雑でファミリー向けでない

### 1.2 提供価値（UVP: Unique Value Proposition）

> 「我が子が主人公になれる絵本を、誰でも5分で作れる。AIが紡ぐ物語と挿絵で、世界にひとつだけの思い出を。」

プロンプト不要・子どもに安全なコンテンツフィルタを備えた、ファミリー特化のAI絵本生成サービス。

### 1.3 ターゲットユーザー

| 項目 | 内容 |
|---|---|
| **メインターゲット** | 0〜6歳の子どもを持つ親（25〜40代） |
| **サブターゲット** | 祖父母・保育士・幼稚園教諭 |
| **ペルソナ概要** | 30代前半・共働き・スマホネイティブ・子どもに特別な体験を与えたい |
| **ユーザーの現在の代替手段** | 市販の絵本を購入 / 絵本作成サービス（しまじろう等） / 手書きで自作 |

### 1.4 MVP機能スコープ（MoSCoW法）

| 優先度 | 機能 | 概要 |
|---|---|---|
| **Must** | AI物語生成 | 子どもの名前・年齢・テーマを入力するとLLMが物語を生成 |
| **Must** | AI挿絵生成 | Replicate API（FLUX Schnell）でページごとに挿絵を生成 |
| **Must** | 絵本ビューア | 見開きページ形式でスマホ上に表示・めくり演出 |
| **Must** | 子ども向けコンテンツフィルタ | 不適切コンテンツのブロック・安全なプロンプト設計 |
| **Should** | テーマテンプレート | 「誕生日」「おやすみ前」等のシーン別テンプレート |
| **Should** | 読み上げ機能 | テキストを音声読み上げ（日本語TTS） |
| **Should** | 物理本印刷注文 | 生成した絵本をA4/A5 冊子として印刷・配送注文 |
| **Could** | キャラクター一貫性設定 | 主人公の外見を固定して複数ページに反映 |
| **Could** | 絵本シェア機能 | SNSへのシェア・家族への送付リンク |
| **Won't（将来）** | 多言語対応 | 英語・中国語での絵本生成 |
| **Won't（将来）** | 絵本コミュニティ | ユーザー間での絵本公開・閲覧 |

#### 競合の強み・弱み

**Story.com（海外AI絵本）:**
- **強み**: AI生成品質が高い、英語圏での先行事例
- **弱み**: 日本語未対応、子ども安全性の明示なし
- **差別化ポイント**: 日本語特化・子ども安全フィルタ・物理本印刷

**Canva:**
- **強み**: デザインテンプレート豊富、ブランド認知度高い
- **弱み**: 全自動生成ではない、ファミリー特化でない
- **差別化ポイント**: 完全自動生成・入力5項目だけで完成

**しまじろう（ベネッセ）:**
- **強み**: 子どもの安全性・ブランド信頼度・既存ユーザー基盤
- **弱み**: コンテンツが固定的・AI生成ではない
- **差別化ポイント**: 子どもの名前・体験を反映したパーソナライズ

#### 市場の空白・機会

1. **日本語特化のAI絵本生成サービスが存在しない** — 海外サービスが先行しているが日本市場は未開拓
2. **物理本印刷×AIの組み合わせが空白** — デジタル生成から実物の絵本へのシームレスな導線がない
3. **0〜3歳向けの完全自動生成ツールがない** — 操作不要・テンプレートなしで親が直感的に使えるサービスが不在

## 5. リスク・懸念事項

### 5.1 技術リスク

| リスク | 影響度 | 対策 |
|---|---|---|
| FLUX Schnellで生成キャラクターの一貫性を保てない | 高 | 当初はページ毎に独立した挿絵として許容、将来的にControlNetやIP-Adapterで改善 |
| 画像生成に時間がかかりUXを損なう | 中 | 非同期生成 + 生成中アニメーション演出・プログレス表示 |
| Replicate APIの障害・価格改定 | 中 | バックアップAPI（fal.ai等）を検討、コスト上限アラート設定 |
| AI生成コンテンツに不適切表現が混入 | 高 | プロンプトの安全設計・出力フィルタ・人手レビュー体制の整備 |

### 5.2 ビジネスリスク

| リスク | 影響度 | 対策 |
|---|---|---|
| 有料転換率が低く収益化に時間がかかる | 高 | 無料枠を意図的に制限（月3冊）・初回生成後に印刷注文訴求 |
| 大手（ベネッセ・Google等）の参入 | 中 | 日本語品質・ファミリーUXの先行優位を早期に確立 |
| 印刷パートナーの品質・配送遅延 | 中 | 複数印刷パートナーを契約・SLA設定・品質チェックフロー整備 |
| 生成コンテンツの著作権問題 | 中 | 利用規約で生成物の権利帰属を明確化、AI生成物の著作権動向を継続監視 |

### 5.3 法務・規制リスク

| リスク | 影響度 | 対策 |
|---|---|---|
| **個人情報保護法**：子どもの名前・画像等の取扱い | 高 | プライバシーポリシー整備・データ最小化原則の適用・データ保存期間の明示 |
| **App Store / Google Playガイドライン**：子ども向けアプリの制限 | 高 | 子ども向けカテゴリのポリシー（COPPA、Apple CIPA準拠）を確認・適合 |
| **AI生成コンテンツの著作権**：挿絵の権利帰属 | 中 | 利用規約で「生成物は利用者に帰属」と明記・Replicateの利用規約との整合確認 |
| **特定商取引法**：物理本販売・有料プランの表記 | 中 | 特商法に基づく表記ページの設置（販売者情報・返品ポリシー等） |
| **景品表示法**：AI生成品質に関するサンプル表示 | 低 | 「生成例はあくまでサンプル」等の注記を明示 |
| **GDPR**：将来のグローバル展開時 | 中 | アーキテクチャ設計時にデータ居住地（日本リージョン）を考慮 |

## 6. 意思決定が必要なポイント
