# UX_DESIGN.md

## 概要
「売り物化」を見据えたUXの整備。不備のある導線や、ユーザーが迷いやすい箇所を改善する。

## 優先改善項目

### 1. 本棚UI / ホーム画面
- **一覧性**: 作成した絵本をカード形式で美しく並べる。
- **空状態 (Empty State)**: 1冊もない場合に、作成を促すメッセージとボタンを表示。
- **削除・管理機能**: 不要な絵本を整理できるUI。

### 2. 絵本閲覧UI
- **ページ送り**: スワイプやボタンによる自然なページ遷移。
- **3D演出**: ページをめくる際のほのかな 3D 表現（CSS `transform`）。

### 3. 生成中進捗表示
- **ステップの可視化**: 「お話を作っています」「絵を描いています」など、現在のフェーズを明確にする。
- **待ち時間の演出**: ユーザーが飽きないよう、面白いヒントや豆知識を表示。

### 4. 失敗・エラー時の導線
- **再試行**: 失敗時に「もう一度作る」ボタンを表示。
- **エラーメッセージ**: 技術的な文言ではなく、ユーザーに寄り添った解決策を提示。

### 5. オンボーディング
- 初回利用時に、アプリの使い方や「えほん」の魅力を伝えるシンプルなステップ。

### 6. 共有・出力 (将来拡張)
- PDF出力や、SNSでの共有URL発行のUI設計。

## 実装スケジュール
- Phase 4 では、まず本棚と生成中のUIを最優先で改善する。
