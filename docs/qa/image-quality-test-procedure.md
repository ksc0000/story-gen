# 絵本スタイル品質テスト 手順書

> バージョン: v1.0  
> 対象リリース: 全スタイル公開前の品質確認  
> 作成日: 2026-06-08

---

## 1. 目的と概要

この手順書は、EhonAI（絵本生成アプリ）が提供する **10種類の絵本イラストスタイル** の品質を、
システムの内部構造を知らない人でも確認できるよう手順化したものです。

### テスト内容

| フェーズ | 内容 | 冊数 |
|---------|------|------|
| Phase 1 | 全スタイル × guided_ai × 4ページ | 10冊 |
| Phase 2 | 主要スタイル（Phase 1結果から選択）× 12ページ | 2〜3冊 |
| **合計** | | **12〜13冊** |

> **備考**: アプリのスタイル定義には技術的に12件のエントリがありますが、
> `watercolor` と `flat` は既存スタイルの別名（エイリアス）のため、
> 実質10種類の固有スタイルが対象です。

### 何を確認するか

- 各スタイルが意図した雰囲気・タッチで絵が生成されるか
- 生成が正常完了するか（エラーなし）
- 4ページ全ページに絵が表示されるか
- 絵本全体として見たとき、品質が一定水準を満たしているか
- （Phase 2）12ページを通してキャラクターの見た目が一貫しているか

## 2. 前提条件

### 2.1 必要なもの

- スマートフォンまたはPC（Chrome / Safari 推奨）
- テスト用アカウント（管理者から入手）
- 本手順書のPrintout または デジタル版
- 記録用のテキストエディタまたは印刷した記録シート

### 2.2 テスト用アカウント情報

> ⚠️ アカウント情報は管理者から別途受け取ってください。
> このドキュメントには記載しません。

| 項目 | 内容 |
|------|------|
| アプリURL | 管理者から入手 |
| メールアドレス | 管理者から入手 |
| パスワード | 管理者から入手 |

### 2.3 管理者が事前に実施すること（テスト開始前・1回のみ）

テスト担当者が始める前に、管理者は以下を完了させてください。

- [ ] テスト用Googleアカウントでアプリにサインインできる状態にする
- [ ] **テストアカウントの月次制限を無効化する**  
      （Firebase コンソール → Firestore → `users/{userId}` → `generationOverride.bypassMonthlyLimit: true`）
- [ ] テスト用の「子どもプロフィール」を1件作成する  
      （名前: `テストちゃん`、年齢: `4才`）
- [ ] テスト担当者にアプリURLとログイン情報を共有する

## 3. 共通テスト入力値

以下の入力値を **全テストケースで共通** で使用します。
毎回同じ値を入力することで、スタイルの違いのみを比較できます。

| 入力項目 | 値 |
|---------|-----|
| 子どもの名前 | テストちゃん（事前作成済みプロフィール） |
| 作成モード | **かんたんカスタム** |
| テーマ | 画面に最初に表示される **おやすみ/寝かしつけ系** のテーマ |
| ページ数 | Phase 1: **4ページ（短い）** ／ Phase 2: **12ページ（長い）** |
| 伝えたいメッセージ | `いつも笑顔でいてね。大好きだよ。` |
| 場所（任意） | `（空白のまま）` |

> **メモ**: テーマ選択は毎回同じテーマを選んでください。
> 「おやすみ」や「ねんね」などのカテゴリにある最初のテーマを一貫して使用します。

## 4. Phase 1：スタイル網羅テスト（10冊）

### 4.1 テストケース一覧

以下10種類のスタイルについて、それぞれ1冊ずつ作成します。

| No. | スタイル名（UI表示） | スタイルID（技術参照用） |
|-----|------------------|---------------------|
| 1 | やさしい水彩 | `soft_watercolor` |
| 2 | ふんわりパステル | `fluffy_pastel` |
| 3 | クレヨンで描いた絵本 | `crayon` |
| 4 | シンプルフラット | `flat_illustration` |
| 5 | わくわくアニメ風 | `anime_storybook` |
| 6 | クラシック絵本 | `classic_picture_book` |
| 7 | ぷっくり3Dトイ風 | `toy_3d` |
| 8 | 紙あそびコラージュ | `paper_collage` |
| 9 | やさしい鉛筆スケッチ | `pencil_sketch` |
| 10 | カラフルポップ | `colorful_pop` |

### 4.2 作成手順（1冊あたり）

以下の手順を **1スタイルにつき1回** 繰り返します。

---

**Step 1: アプリを開いてログインする**

1. ブラウザでアプリURLを開く
2. 「Googleでサインイン」をタップ
3. テスト用アカウントでログイン

---

**Step 2: 新しい絵本を作成する**

1. ホーム画面で「絵本を作る」または「＋」ボタンをタップ
2. 子どものプロフィール選択画面が表示されたら、「テストちゃん」を選ぶ

---

**Step 3: テーマと作り方を選ぶ**

1. 「作り方・テーマを選ぶ」画面が表示される
2. 画面上部の作り方から **「かんたんカスタム」** を選ぶ
3. テーマ一覧から、**おやすみ/寝かしつけ系の最初のテーマ**（毎回同じもの）を選ぶ
4. 「次へ」をタップ

---

**Step 4: ストーリー内容を入力する**

1. 入力画面が表示される
2. **ページ数** を **「短い（4ページ）」** に設定する  
   ※ ページ数の選択肢が表示されていない場合は既定のまま進める（メモに記録）
3. **「伝えたいメッセージ」** 欄に以下を入力する：  
   ```
   いつも笑顔でいてね。大好きだよ。
   ```
4. その他の入力欄は **空白のまま** にする
5. 「次へ」または「スタイルを選ぶ」をタップ

---

**Step 5: イラストスタイルを選ぶ**

1. スタイル選択画面が表示される
2. 今回テストするスタイル（テストケース一覧のNo.順）を選択する
3. スタイルが選択されたことを確認し（ハイライト表示）、「絵本を作る」をタップ

---

**Step 6: 生成の完了を待つ**

1. 「生成中」画面が表示される（通常 2〜5分程度かかります）
2. 画面をそのままにして待つか、アプリをバックグラウンドに置いて後で確認する
3. 完了通知またはホーム画面の絵本一覧で完了を確認する

> **タイムアウトについて**: 10分以上「生成中」のまま動かない場合は、
> 管理者に連絡してください。ページ番号とスタイル名をメモしておく。

---

**Step 7: 生成結果を確認して記録する**

絵本が完成したら、以下を確認して §7「Phase 1 結果記録シート」に記入してください。

1. ホーム画面から完成した絵本をタップして開く
2. 表紙〜全ページをスワイプで確認する
3. 記録シートの該当行を埋める

---

### 4.3 Phase 1 確認ポイント（1冊ごと）

各絵本を開いた後、以下の視点で確認します：

**A. 基本チェック（◯/☓）**
- [ ] 絵本が正常に完成しているか（エラー表示がないか）
- [ ] 4ページ全ページに画像が表示されているか
- [ ] テキスト（物語文）が日本語で正常に表示されているか

**B. スタイル再現度（1〜5点で評価）**
- 選んだスタイル名が表す雰囲気が絵に出ているか
  - 5: 完全に一致している
  - 4: おおむね一致している
  - 3: 雰囲気は出ているが弱い
  - 2: ほとんど感じられない
  - 1: まったく異なる印象

**C. 総合品質（1〜5点）**
- 商品として人に見せられる品質か
  - 5: 十分満足できる品質
  - 4: 概ね良好
  - 3: 惜しいが使える
  - 2: 改善が必要
  - 1: リリース不可

## 5. Phase 2：キャラクター一貫性テスト（2〜3冊）

### 5.1 テストの目的

12ページ（長い版）で生成したとき、絵本を通してキャラクターの外見が
どの程度一貫しているかを確認します。

### 5.2 対象スタイルの選定（Phase 1 終了後に記入）

Phase 1 の結果をふまえて、以下の基準でスタイルを選んでください。

| 選定基準 | 選定するスタイル |
|---------|--------------|
| Phase 1 で総合品質が最も高かったスタイル | _______________________ |
| 人物描写が最も特徴的だったスタイル | _______________________ |
| 疑問点や改善余地があり詳しく確認したいスタイル（任意） | _______________________ |

> **推奨候補（事前選定）**: やさしい水彩・わくわくアニメ風・クレヨンで描いた絵本  
> Phase 1 の結果によって変更してください。

### 5.3 作成手順

Phase 1 と同じ手順（Step 1〜7）を繰り返しますが、**Step 4 のみ以下を変更**します：

> **Step 4 変更点：ページ数を「長い（12ページ）」に設定する**

その他の入力値は Phase 1 と完全に同一にしてください。

### 5.4 Phase 2 確認ポイント

各ページを順番に確認して、以下を評価します。

**D. キャラクター一貫性（ページ間比較）**

全12ページの画像を並べて（またはスワイプして）確認し、以下を評価する：

- 主人公（テストちゃん）の顔・体型・服装が一貫しているか
- ページをまたいで同じキャラクターと認識できるか
- 特に **表紙・中盤（6ページ目）・最終ページ** の3点で比較する

| ページ比較 | 評価（1〜5） | 気になった点 |
|-----------|-----------|------------|
| 表紙 → 1ページ目 | | |
| 表紙 → 中間ページ | | |
| 表紙 → 最終ページ | | |

## 8. 問題発生時の対応

### よくある問題

| 現象 | 対処方法 |
|------|---------|
| 生成が10分以上終わらない | 管理者に連絡。テストケースNo.とスタイル名をメモしておく |
| 「エラーが発生しました」と表示 | スクリーンショットを撮影し管理者に共有。次のテストケースへ進む |
| 一部のページだけ画像が表示されない | 記録シートの「全ページ表示」欄に「一部」と記入。画像のないページ番号もメモ |
| スタイル選択肢に一部スタイルが見当たらない | 管理者に連絡 |
| ページ数「4ページ」が選べない | 管理者に連絡（プラン設定の確認が必要な可能性） |

### 記録・エスカレーション

問題が発生した場合は、以下の情報を管理者に共有してください：

1. テストケースNo.とスタイル名
2. 発生した現象（スクリーンショット推奨）
3. 手順のどのStepで発生したか
