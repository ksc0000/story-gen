# IMG-003: Background Removal Evaluation for Child Reference Images

## 1. 現状確認 (fixed-first-zoo テスト結果)
`docs/IMG-002_REFERENCE_VERIFICATION_REPORT.md` (2026-05-12) に基づく現状：
- **ステータス:** プロンプト側の防御（isolation suffix）により、sandbox/playground のリークは概ね抑制されている。
- **課題:** 小さな看板状のアーティファクトが残るなど、参照画像の背景情報が完全に無視されるわけではない。構造的な堅牢性を高めるため、画像そのものの背景除去が有効である。

## 2. 背景除去ライブラリ・APIの比較調査

| 候補 | 方式 | プライバシー | 精度 | コスト | 処理速度 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **rembg** | ローカル (Python) | ✅ **最高** (外部送信なし) | 中〜高 (BiRefNet/U2Net) | ✅ **無料** | サーバー依存 (1-5秒) |
| **remove.bg API** | 外部API | ⚠️ 要同意 (Canva系) | ✅ **最高** (高解像) | ⚠️ 有料 (1枚5-10円) | ✅ 高速 (< 1秒) |
| **Cloudinary** | 外部API | ⚠️ 要同意 (Cloud) | ✅ **最高** (高解像) | ⚠️ 有料 (従量課金) | ✅ 高速 (< 1秒) |
| **Firebase ML Kit** | オンデバイス | ✅ 高 (デバイス処理) | 中 (Selfieに特化) | ✅ 無料 | ✅ リアルタイム |

### 調査詳細
- **rembg:** 最新の `BiRefNet` モデルを使用することで、髪の毛などの細かい切り抜き精度が大幅に向上している。Node.js からは Python シェル経由または HTTP サーバーモード（Docker）で呼び出す必要がある。
- **remove.bg / Cloudinary:** 商業レベルの最高精度だが、規約上「子ども向けサービス」を明示的に対象外としている場合があり（例：Kaleido/remove.bg）、13歳未満の写真を扱うための COPPA 対応や個別同意フローの構築コストが高い。
- **Firebase ML (Kit):** 自撮り（Selfie）に特化しており、全身像や手持ちのおもちゃが含まれる「子どものスナップ写真」では精度が不足するリスクがある。

## 3. プライバシー・同意ポリシーの評価
- **ローカル処理 (rembg):**
  - 画像データがプロジェクトの境界（GCPプロジェクト内）を出ないため、COPPA/GDPR上の「データ受信者」が増えず、最も安全。
  - 同意フローも「AIによる人物抽出」のみに絞れる。
- **外部API:**
  - 画像がサードパーティサーバーに一時保存される。
  - AI学習への利用をオプトアウトする設定が必須。
  - 利用規約に「13歳未満の利用禁止」が含まれていることが多く、B2Cの子ども向けサービスでの利用には法務的ハードルがある。

## 4. 精度上の注意点（子ども写真特有のリスク）
- **髪の毛の輪郭:** 子どもの細い髪の毛は背景に溶け込みやすく、雑なモデルでは「ヘルメット状」に切り取られる。`alpha matting` 機能を備えたモデル（rembg 等）が必要。
- **ぬいぐるみ・おもちゃ:** 子どもが手に持っているおもちゃが「背景」と判定され、体の一部が欠損するリスク。
- **多様な服装:** 透け感のあるレースの服や、背景色に近い服（例：芝生で緑の服）での誤判定。
- **対策:** 失敗（前景が極端に小さい、または抽出失敗）した場合は、背景除去済みの画像ではなく「元の画像」を使用するフォールバックが必須。

## 5. PoC 設計案：採用候補「rembg」

### 前処理のタイミング
- **非同期アップロード時:** ユーザーが子どもの写真をアップロードした直後、Firebase Functions (onObjectFinalized) で背景除去を実行。生成リクエスト時のレイテンシを最小化する。

### データの保存・キャッシュ
- Cloud Storage 上に以下の構成で保存：
  - `originals/{childId}.jpg` (元画像：万が一の再処理用)
  - `neutrals/{childId}.png` (背景除去済み：生成に使用)
- `childProfiles` ドキュメントに `neutralReferenceImageUrl` フィールドを追加。

### 失敗時のフォールバック
1. 背景除去プロセスでエラー、または出力画像が空（または閾値以下のサイズ）の場合：
   - `neutralReferenceImageUrl` を null のままにする。
2. 絵本生成 (`generate-book`) 時：
   - `neutralReferenceImageUrl` があればそれを使用。
   - なければ従来の `referenceImageUrl` を使用し、既存のプロンプトによる防御（isolation suffix）に頼る。
